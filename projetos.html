O NexDoc é uma aplicação de gestão de contratos pensada para quem precisa de criar, assinar e arquivar documentos sem depender de conta ou de servidor — tudo funciona diretamente no browser.

Dá para começar um contrato do zero ou importar um ficheiro existente (TXT, HTML ou DOCX), editá-lo num editor de texto rico com formatação, títulos, listas e hiperligações, e depois assiná-lo à mão diretamente no ecrã. Cada documento fica selado com um hash SHA-256 que garante a integridade do conteúdo, e a exportação final é feita em PDF através do diálogo de impressão do browser.

Funcionalidades principais

  • Gestão do ciclo de vida do contrato — rascunho, pendente de assinatura, assinado — com painel de estatísticas e arquivo pesquisável por estado, data ou valor.
  • Mais de 50 moedas de África, Europa, Américas e Ásia, com pesquisa por nome, código ou símbolo.
  • Alertas automáticos quando um contrato está a 7 dias ou menos de expirar.
  • Logótipo personalizado no cabeçalho dos PDFs.
  • Backup e restauro dos contratos em JSON.
  • Interface em quatro idiomas — Português, Inglês, Francês e Espanhol — e tema claro/escuro.

O que o torna diferente

Corre inteiramente no browser e todos os dados ficam guardados no próprio dispositivo, sem passar por nenhum servidor externo. A importação de ficheiros DOCX depende de uma biblioteca carregada por CDN, por isso a primeira importação pode exigir ligação à internet.

Limitações conhecidas

  • O editor ainda não suporta inserção de tabelas.
  • Sem gestos de swipe nem otimizações específicas para teclado virtual — o toque está implementado apenas no quadro de assinatura.
  • A exportação em PDF depende do diálogo de impressão nativo do browser, exigindo a escolha manual de "Guardar como PDF".

NexDoc is a contract-management app built for creating, signing and archiving documents without needing an account or a server — everything runs directly in the browser.

You can start a contract from scratch or import an existing file (TXT, HTML or DOCX), edit it in a rich text editor with formatting, headings, lists and hyperlinks, then sign it by hand right on the screen. Every document is sealed with a SHA-256 hash that guarantees content integrity, and the final export is done as a PDF through the browser's print dialog.

Key features

  • Full contract lifecycle management — draft, pending signature, signed — with a stats dashboard and a searchable archive filterable by status, date or value.
  • Over 50 currencies from Africa, Europe, the Americas and Asia, searchable by name, code or symbol.
  • Automatic alerts when a contract is 7 days or less from expiring.
  • Custom company logo on the PDF header.
  • Backup and restore of contracts as JSON.
  • Interface in four languages — Portuguese, English, French and Spanish — plus light/dark theme.

What makes it different

Runs entirely in the browser, with all contract data stored on the device itself — nothing goes to an external server. DOCX import relies on a CDN-loaded library, so the first import may require an internet connection.

Known limitations

  • The editor doesn't yet support inserting tables.
  • No swipe gestures or virtual-keyboard-specific optimizations — touch interaction is only implemented on the signature pad.
  • PDF export relies on the browser's native print dialog, requiring manually choosing "Save as PDF".

O LuxeStay é uma PWA de reservas de hotel com fluxo completo de descoberta, reserva e gestão de estadias — construída inteiramente em JavaScript vanilla e CSS, sem frameworks no frontend, com Supabase como backend.

O objetivo do projeto foi testar até onde é possível ir com HTML, CSS e JavaScript puros sem abrir mão de autenticação real, dados persistidos (quartos, avaliações, reservas) e um fluxo de reserva completo — da pesquisa ao pagamento simulado — tudo isto com interface bilingue PT/EN.

Funcionalidades principais

  • Autenticação completa — registo, login, recuperação de senha, validação de idade mínima e força de senha.
  • Pesquisa e filtro de quartos por tipo, disponibilidade e favoritos.
  • Galeria de fotos por quarto com lightbox em ecrã inteiro.
  • Fluxo de reserva: seleção de datas → dados do hóspede → pagamento simulado (transferência bancária ou QR code) → confirmação.
  • Secção "Minhas Estadias" com histórico, cancelamento e estado vazio dedicado.
  • Avaliações por quarto, ligadas ao perfil do autor.
  • Interface bilingue PT/EN, com o conteúdo dos quartos traduzido diretamente na base de dados.
  • Instalável como app e funcional offline para páginas já visitadas.

O que o torna diferente

Usa Supabase (Postgres + autenticação + Row Level Security) como backend, o que permitiu concentrar o tempo em UX e regras de negócio em vez de reconstruir autenticação e infraestrutura do zero. Todas as tabelas sensíveis têm políticas de RLS ativas, garantindo que cada utilizador só acede às suas próprias reservas — mesmo com a chave pública exposta no cliente.

É uma PWA instalável em vez de uma app nativa: um Service Worker aplica estratégias de cache diferentes por tipo de recurso (páginas em network-first, imagens em cache-first), cobrindo o caso de uso principal — revisitar o app com carregamento rápido — sem a complexidade de publicar em lojas de aplicações.

O pagamento é simulado (transferência bancária ou QR code fictício), já que o foco do projeto não é a integração real com um gateway de pagamentos.

LuxeStay is a hotel-booking PWA with a full discovery, booking and stay-management flow — built entirely in vanilla JavaScript and CSS, with no frontend framework, using Supabase as the backend.

The goal of the project was to see how far plain HTML, CSS and JavaScript could go without giving up real authentication, persisted data (rooms, reviews, bookings) and a complete booking flow — from search to mocked payment — all wrapped in a bilingual PT/EN interface.

Key features

  • Full authentication — sign-up, login, password recovery, minimum-age validation and password strength checks.
  • Room search and filtering by type, date availability and favorites.
  • Photo gallery per room with a fullscreen lightbox.
  • Booking flow: date selection → guest details → mocked payment (bank transfer or QR code) → confirmation.
  • "My Stays" section with booking history, cancellation and a dedicated empty state.
  • Per-room reviews linked to the author's profile.
  • Bilingual PT/EN interface, with room content translated directly in the database.
  • Installable as an app and functional offline for already-visited pages.

What makes it different

Uses Supabase (Postgres + auth + Row Level Security) as the backend, which kept the focus on UX and business logic instead of rebuilding authentication and infrastructure from scratch. Every sensitive table has active RLS policies, ensuring each user can only access their own bookings — even with the public key exposed on the client.

It's an installable PWA rather than a native app: a Service Worker applies different caching strategies per resource type (pages network-first, images cache-first), covering the main use case — revisiting the app with fast load times — without the complexity of publishing to app stores.

Payment is mocked (bank transfer or a fictional QR code), since real payment-gateway integration wasn't the goal of the project.

An interactive HTML, CSS and JavaScript ebook — no frameworks, no build step — that teaches the Linux terminal from scratch. It works as a single-page reader, with chapter navigation, a side index, a progress bar and full Portuguese/English support.

Features

  • Bilingual — instant switching between Portuguese and English, including the comments inside code blocks.
  • Light and dark theme.
  • Resume reading — locally saves which page, language and theme you were on, and returns exactly there the next time you open the file.
  • Direct per-page links, letting the guide open straight into a specific chapter.

Current content

The guide currently covers Chapter 1 — Navigation Fundamentals, with 46 commands and combinations organized into six thematic blocks: finding out where you are, listing and viewing content, navigating strategically, finding and searching, structure and metadata, and advanced combinations. New chapters are being added progressively.
